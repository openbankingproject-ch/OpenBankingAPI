# <span style="color: #253165">Open API Customer Relationship - Master Implementation Roadmap</span>

## Executive Summary

This master roadmap consolidates all implementation timelines of the Open API Customer Relationship project and provides a unified source for project phases, milestones, and deliverables. The implementation follows a three-phase approach over 36 months, from the foundation phase to the production rollout.

**<span style="color: #F85F3D">Strategic Goal:</span>** Establish Switzerland as a leading market for customer data portability and financial service integration through secure, standardized APIs. The Open Banking Project provides an open interface for the customer relationship, which includes onboarding, maintenance of customer data, and balancing of a customer relationship. The interface is designed for private customers, businesses, and corporate clients.

The Open API Customer Relationship defines a basic set of customer data that enables identification and the acquisition of services in the network. Through the digital portability of identified customer data, the seamless and uninterrupted use of services becomes easier, more efficient, and safer for all participants.

**Reference Process Building Blocks for Ecosystem Connectivity:** 
- **Initialization:** Service development via corresponding channel
- **Product Selection:** Ensuring connectivity for the purchase of services in an ecosystem
- **Self-declaration:** Tax and regulatory aspects
- **Basic Data:** Personal details and address/contact data (E-ID compatible)
- **Extended Data:** Ensuring cross-industry service selection
- **ID Document:** Presentation of the identification document
- **Identification:** Verification of identity using regulatory compliant procedures
- **Check:** Execution of regulatory necessary checks (age check, PEP, sanctions lists)
- **Consent:** Consumer consent in the context of data protection and consent management
- **Signature:** Confirmation using regulatory required signature level
- **Backoffice:** Ensuring connection to processing

**Consent and Security Flow:**
The consent flow maps the customer's approval, while the security flow is formulated independently, thereby ensuring connectivity into any ecosystem.

**Standards and Compatibility:**
Established standards are used to ensure international compatibility. Coordination and compatibility with other initiatives is guaranteed:
- SFTI: Customer data usage in the payment traffic API
- Open Wealth: Customer data usage in the Customer Management API for EVV connection
- eCH: Use of customer data in the governmental context

**Availability:**
The open interface will be publicly available after completion, including the I14Y platform of the Federal Statistical Office. An ISO accreditation of the Open API Customer Relationship is aspired to.

## <span style="color: #0070C0">Implementation Roadmap</span>

### <span style="color: #4cb867ff">Phase 1: Foundation (until 06/26)</span>
**Conceptual Basics:**
- Market analysis, requirements, reference processes and data
- API Design, federated network, Consent & Security Flow
- Legal framework, test concept

**Alpha Version Provisioning:**
- Functionality onboarding and maintenance for Swiss and EU citizens with Swiss domicile
- Community-based further development and verification
- Publication of the tested and verified interface

**Network Basics:**
- Governance for federated structures
- Regulatory clarifications
- MVP implementation "Identification Private Customers" with network transfer

### <span style="color: #FFC000">Phase 2: Expansion (from 07/26)</span>
**Further Development:**
- Conception of federated system governance
- Expansion of the MVP "Identification Private Customers" with 5-10 additional companies
- Launching further MVPs
